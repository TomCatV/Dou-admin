# Dou 商用多租户后台使用手册

最后更新：2026-06-02

## 入口与账号

后台统一入口为 `Dou-admin` 管理端。平台管理员和圈主账号共用同一登录页，登录后系统会根据账号角色展示对应菜单。

新建账号或重置密码后的账号首次登录必须先修改密码。修改密码入口在右上角头像下拉菜单；强制改密时系统会自动跳转到修改密码页。密码修改成功后旧登录态会失效，需要重新登录。

## 角色总览

| 角色         | 使用对象       | 主要范围                                                       |
| ------------ | -------------- | -------------------------------------------------------------- |
| 超级管理员   | 平台唯一主账号 | 全平台、账号派发、分组授权、SaaS 套餐、财务和安全              |
| 1 级管理员   | 高权限平台运营 | 平台治理、圈主账号、交易资金、套餐运营，不能替代唯一超级管理员 |
| 2 级管理员   | 运营负责人     | 用户、圈子、内容、售后、提现和部分圈主账号管理                 |
| 3 级管理员   | 客服/审核      | 举报、审核、只读数据和低风险运营动作                           |
| 圈主账号     | 付费圈主       | 自己绑定圈子的经营、成员、资源、订单和私域工具                 |
| 圈主子账号   | 圈主员工/助理  | 受圈主授权的运营工作                                           |
| 圈主只读账号 | 财务/观察者    | 查看数据，不执行写入和资金动作                                 |

## 超级管理员

超级管理员只有一个账号，负责搭建平台管理边界。

常用流程：

1. 进入 `账号权限 / 后台账号`，创建 1-3 级管理员或圈主账号。
2. 创建圈主账号时选择 `指定圈子`，绑定圈子 owner 对应的小程序用户。
3. 进入 `账号权限 / 账号分组`，按套餐、托管客户、内测客户等方式创建分组。
4. 对需要查看主房间聊天记录的分组开启授权，再把对应圈主账号加入该分组。
5. 进入 `SaaS计费 / 套餐订阅`，维护套餐、开通租户、登记订阅订单和到期时间。
6. 进入 `系统安全 / 操作日志`，查看敏感操作记录。

主房间聊天记录授权规则：只有超级管理员能维护账号分组和开关。圈主账号加入已授权分组后，只能查看自己绑定圈子的主房间记录，不能跨圈查看。

## 1-3 级管理员

1 级管理员适合给核心运营负责人使用，可处理大部分运营、交易和套餐相关工作。2 级管理员适合内容运营、客服负责人。3 级管理员适合审核和客服人员。

权限建议：

| 级别 | 建议职责                                         |
| ---- | ------------------------------------------------ |
| 1 级 | 派发圈主账号、处理治理和资金、配置套餐、协助续费 |
| 2 级 | 管理用户、圈子、资源卡、售后和提现               |
| 3 级 | 查看数据、处理举报、人工复核和低风险客服动作     |

管理员不能创建或提升超过自己权限边界的账号。涉及停用账号、重置密码、资金审核、内容下架等动作时，系统会保留操作日志。

## 圈主账号

圈主账号由平台派发，必须绑定微信用户和指定圈子。圈主登录后主要使用以下模块：

| 模块                  | 用途                                                   |
| --------------------- | ------------------------------------------------------ |
| 圈主后台 / 圈主工作台 | 查看成员、资源卡、订单、收入、套餐状态和当前用量       |
| 圈主后台 / 店铺资料   | 维护买家看到的店铺资料、展示设置和房间信息             |
| 圈主后台 / 圈内成员   | 搜索成员、查看身份和加入时间                           |
| 圈主后台 / 圈内资源   | 查看资源卡售卖状态、购买数和收入                       |
| 私域转化 / 私域线索   | 维护潜客、意向等级、标签和跟进状态                     |
| 私域转化 / 转化工具   | 设置欢迎自动化、优惠券、邀请码和活动                   |
| 私域转化 / 转化漏斗   | 查看曝光、入圈、激活、成交、复购、裂变数据             |
| AI 经营 / 经营助手    | 生成经营日报、活动文案，查看 AI 历史记录并人工确认建议 |
| 交易资金 / 订单售后   | 查看本圈订单和售后                                     |
| 交易资金 / 钱包提现   | 查看余额并发起提现申请                                 |

套餐到期后，平台可把租户状态调整为到期或暂停。系统保留数据，后续可通过续费订单恢复 active 状态。

### 圈主售后处理

进入 `交易资金 / 订单售后`，切换到 `售后` 页签后可以查看本圈售后单。

可执行动作：

1. `发送回复`：与买家协商，回复会同步到售后消息。
2. `补发资源`：用于网盘链接、资料说明等资源重新交付；只记录动作，不改变资金。
3. `更换卡密`：仅卡密交付商品可用；系统会禁用旧卡密并发放一条新卡密，库存不足时需要先回到商品中心补库存。
4. `通过并退款`：发起微信原路退款，退款成功与否以微信回调或查单为准。
5. `刷新退款`：当退款处理中或回调延迟时，手动查询最新退款状态。

所有售后动作会进入售后动作轨迹和操作日志。圈主不能直接修改订单支付状态，也不能手动把退款标为成功。

## 圈主子账号和只读账号

圈主子账号适合运营助理使用，默认可查看和维护本圈基础运营数据。圈主只读账号适合财务、合伙人或观察者，只建议授予查看权限。

子账号使用边界：

1. 只能访问绑定圈子的后台数据。
2. 不能访问平台全量用户、全量财务或其他圈子。
3. 资金、账号、敏感内容操作应由圈主主账号或平台管理员处理。

## SaaS 套餐与订阅

平台管理员进入 `SaaS计费 / 套餐订阅` 后可以完成：

1. 创建或编辑套餐，配置月付价、年付价、用量限制和功能开关。
2. 给指定圈子开通租户，设置套餐、试用时间、付费到期时间和租户状态。
3. 给租户登记订阅订单，填写计费周期、金额、合同编号、发票抬头和服务到期时间。
4. 查看租户用量，包括成员、子账号、资源卡、线索和订单数量。

套餐字段目前以 JSON 保存用量限制和功能开关，便于后续扩展成更细的功能管控。

## 私域转化工具

当前后台已经支持一套基础私域转化闭环：

| 能力       | 说明                                                 |
| ---------- | ---------------------------------------------------- |
| 线索管理   | 记录姓名、手机号、微信号、来源、意向等级、状态和备注 |
| 标签体系   | 给成员或线索打标签，支持分层运营                     |
| 欢迎自动化 | 新人进圈后的欢迎语、任务和优惠券引导                 |
| 优惠券     | 用于首购、复购和活动转化                             |
| 邀请码     | 区分渠道、负责人、使用次数和奖励说明                 |
| 转化活动   | 支持报名、优惠、挑战、直播课、训练营等场景           |
| 漏斗看板   | 用于复盘曝光到裂变的转化效果                         |

### 优惠券和邀请码成交

圈主进入 `私域转化 / 转化工具` 后，可以创建优惠券和邀请码。

使用方式：

1. 创建优惠券，填写名称、编码、优惠金额、最低消费和总量。
2. 创建邀请码，填写编码、渠道、负责人、最大使用次数和奖励说明。
3. 在优惠券或邀请码表格点击 `复制`，得到带参数的商品链接模板。
4. 把链接中的 `{商品KEY}` 替换为实际商品链接里的商品 KEY，再发给私域用户。
5. 买家打开 H5 商品页后，系统会自动带入券码或邀请码，也可以手动输入后点击 `试算优惠`。
6. 买家下单联系方式当前只支持 `手机号 / QQ号 / 邮箱` 三类，页面会在提交前校验格式，并尽量使用浏览器自动记忆方便复填。
7. 支付成功后，圈主可在 `交易资金 / 订单售后` 查看优惠抵扣和成交来源。
8. `私域转化 / 转化漏斗` 会展示归因成交、优惠订单、邀请码订单、优惠抵扣和归因成交额。

注意：优惠券只影响本次订单应付金额，不修改商品原价。商品改价、优惠券失效或邀请码超限后，旧草稿会提示重新下单。未支付订单不会增加优惠券或邀请码使用次数。

## AI 经营助手

圈主进入 `AI 经营 / 经营助手` 后，可以使用两类能力：

1. `经营日报`：选择最近 1-30 天的数据范围，系统会基于订单、成交额、优惠、售后、商品、线索和买家风险的聚合脱敏数据生成日报。
2. `活动文案`：填写活动目标、人群和语气，系统会基于公开商品字段、优惠券和转化汇总生成候选文案。

使用边界：

1. AI 不会自动修改价格、库存、权限、退款、钱包、优惠券或订单状态。
2. AI 不会接收主房间私聊明细、卡密明文、支付密钥、完整 openid、完整手机号、邮箱、JWT 或管理员密码。
3. 所有生成都会进入历史记录，包含模型、时间、数据范围、状态、失败原因和结构化结果。
4. `人工确认` 只记录运营已采纳该建议，不会自动执行任何业务动作；真正执行仍需回到对应业务页面手动操作。
5. 如果页面提示 `服务端尚未配置 OPENAI_API_KEY`，说明生产环境还没填 AI 密钥；此时可查看历史和兜底摘要，但不能得到真实 AI 结果。

权限说明：圈主主账号和圈主运营子账号默认可生成；圈主只读账号默认只能查看历史。平台如需关闭某个圈主账号的生成能力，可在账号权限中移除 `tenant:ai:generate`。

## 买家风控

超级管理员或 1 级管理员进入 `平台治理 / 买家风控` 可以维护买家风险档案。

风险等级：

| 等级   | 影响                                 |
| ------ | ------------------------------------ |
| 正常   | 不阻断下单，仅作为档案留存           |
| 观察   | 不阻断下单，用于客服重点关注         |
| 黑名单 | 阻断 H5 创建订单草稿和草稿转正式订单 |

新增档案时至少填写 `买家用户ID` 或 `下单联系方式` 之一。联系方式保存后只展示掩码，系统用哈希做匹配。`封禁到期` 留空表示长期阻断；填写到期时间后，到期后不再阻断。

## 安全与审计

登录安全策略：

1. 连续登录失败会触发锁定。
2. 登录日志记录 IP、UA、成功或失败原因。
3. 新建和重置密码后强制首次改密。
4. 修改密码后旧 token 立即失效。
5. 被停用账号无法继续登录。

## 上线前必填配置

生产上线前需要由平台负责人或运维补齐以下配置；未补齐时对应能力应保持关闭或只做本地验证。

| 类别     | 必填项                                                                                         | 说明                                                                  |
| -------- | ---------------------------------------------------------------------------------------------- | --------------------------------------------------------------------- |
| 后台账号 | `ADMIN_BOOTSTRAP_USERNAME`、`ADMIN_BOOTSTRAP_PASSWORD`                                         | 首次种子管理员；上线后立刻改密                                        |
| H5 域名  | `VITE_SHOP_BASE_URL`、`VITE_SHOP_API_BASE_URL`                                                 | 买家公开商品页和公开 API 地址                                         |
| 微信支付 | `WECHAT_NATIVE_PAY_ENABLED`、商户号、证书序列号、私钥、平台证书、API V3 Key、支付/退款通知 URL | 未完成真实商户配置前不要开放微信扫码支付                              |
| 支付宝   | `ALIPAY_PAY_ENABLED`、AppID、应用私钥、支付宝公钥、通知 URL                                    | 应用私钥/公钥建议使用 PEM；未完成真实商户配置前不要开放支付宝扫码支付 |
| 平台费率 | `PLATFORM_TRADE_FEE_BPS`                                                                       | 默认交易技术服务费，后台费率策略会覆盖新订单                          |
| 人工调整 | `PLATFORM_REVENUE_ADJUST_MAX_AMOUNT`                                                           | 平台营收人工调整单次金额上限，默认 100000 分                          |
| 结算提现 | `CREATOR_SETTLEMENT_ENABLED`、`CREATOR_WITHDRAW_ENABLED`、冻结天数、最低提现金额               | 影响圈主钱包和提现                                                    |
| 售后策略 | 售后 SLA、高频退款阈值、已提现订单退款策略                                                     | 默认售后 24 小时；已提现订单建议走平台人工退款                        |
| 风控策略 | 黑名单维护负责人、封禁到期规则                                                                 | 黑名单会阻断 H5 新下单                                                |
| 私域转化 | 优惠券默认上限、邀请码和活动使用规范                                                           | 避免圈主误发无限额优惠                                                |
| AI 助手  | `OPENAI_API_KEY`、`OPENAI_API_PROTOCOL`、`AI_DEFAULT_MODEL`、套餐每日额度、数据留存期           | P5 开放前必填；由 Dou-Server 出网，不依赖 Admin 浏览器 VPN            |

AI 经营助手必填建议：

| 配置                    | 是否必填 | 建议值/说明                                      |
| ----------------------- | -------- | ------------------------------------------------ |
| `OPENAI_API_KEY`        | 生产必填 | 只放 Dou-Server 环境变量，不写入前端、不提交仓库 |
| `OPENAI_API_PROTOCOL`   | 建议填   | 官方 OpenAI 用 `responses`；中转站不确定用 `auto`；只支持 Chat Completions 用 `chat_completions` |
| `OPENAI_BASE_URL`       | 按需填   | 第三方 OpenAI 兼容中转站 baseurl，例如 `https://relay.example.com/v1` |
| `OPENAI_RESPONSES_URL`  | 按需填   | 中转站提供完整 `/responses` 地址时填写            |
| `OPENAI_CHAT_COMPLETIONS_URL` | 按需填 | 中转站提供完整 `/chat/completions` 地址时填写     |
| `AI_DEFAULT_MODEL`      | 必填     | 默认 `gpt-5.5`；上线前按 OpenAI 官方模型文档复核 |
| `AI_DAILY_LIMIT_BASIC`  | 必填     | 入门套餐建议 3-5 次/天                           |
| `AI_DAILY_LIMIT_PRO`    | 必填     | 专业套餐建议 30 次/天                            |
| `AI_REQUEST_TIMEOUT_MS` | 建议填   | 默认 30000；超时只影响 AI 结果                   |
| `AI_MAX_OUTPUT_TOKENS`  | 建议填   | 默认 1800；用于控制成本和响应长度                |

线上排障口径：

1. Admin 页面只调用 Dou-Server，不直接访问 OpenAI；因此线上 AI 助手不应该要求运营电脑开 VPN。
2. 如果页面生成失败，先看 Dou-Server 日志里的 `[ai-business] generation failed`。
3. 如果日志提示 `OpenAI 返回内容不是可解析 JSON`，优先检查中转站协议是否配置正确；可尝试 `OPENAI_API_PROTOCOL=auto` 或 `chat_completions`。
4. 如果日志提示未配置 `OPENAI_API_KEY`，补服务端环境变量并重启 Dou-Server。

运营建议：

1. 超级管理员只保留给实际负责人，不用于日常客服。
2. 1-3 级管理员按职责最小授权。
3. 圈主账号必须绑定正确微信用户和圈子。
4. 主房间聊天记录查看只给明确付费或托管客户分组授权。
5. 定期查看操作日志和登录日志。

## 常见问题

| 问题                             | 处理方式                                                                                                                         |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| 数据库改了密码但旧密码还能登录   | 使用后台修改密码或重置密码接口，系统会更新密码版本并让旧 token 失效                                                              |
| 圈主看不到菜单                   | 检查账号角色是否为 `tenant_owner`、`tenant_staff` 或 `tenant_viewer`，并确认绑定了圈子                                           |
| 圈主无法查看主房间聊天记录       | 超级管理员需要先把圈主账号加入已授权分组                                                                                         |
| 资源卡改价后付款金额不一致       | 服务端已在下单和支付前同步最新资源卡价格，旧待支付订单会刷新金额并重建预支付                                                     |
| 套餐到期后怎么恢复               | 在 SaaS 计费中给租户登记续费订单，系统会更新套餐和 `paid_until`                                                                  |
| 黑名单买家仍能付款               | 检查是否已部署后端 `040` 迁移，并确认买家下单联系方式与风控档案匹配                                                              |
| 支付宝生成二维码时报私钥格式错误 | 检查 `ALIPAY_PRIVATE_KEY` 或 `ALIPAY_PRIVATE_KEY_PATH` 是否为 PEM 格式，建议使用 `-----BEGIN PRIVATE KEY-----` 开头的 PKCS8 私钥 |
| 买家投诉入口在哪里               | 已支付订单页只展示 `进入投诉页` 按钮；点击后会跳到独立投诉页填写问题类型和说明                                                   |
