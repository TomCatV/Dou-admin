# 圈主后台账号派发优化设计

最后更新：2026-06-08（Asia/Shanghai）

## 目标

优化 `账号权限 / 后台账号` 中的圈主账号开通体验，让平台侧发号逻辑更贴近真实业务：

1. 圈主主账号不再手填账号名，而是从已注册小程序用户中远程搜索选择。
2. 圈主主账号登录名固定使用对应用户的 `dxq_id`。
3. 圈主主账号右上角昵称优先展示该用户当前的小程序昵称；未设置时回退到系统默认昵称 `抖小圈用户`。
4. 圈主主账号默认可经营绑定用户自己名下的全部活跃圈子，不再强制绑定单个 `scope_circle_id`。
5. 圈主主账号、圈主子账号、圈主只读账号采用不同的预设权限。
6. 保留“指定圈子”模型给子账号和只读账号使用，不再把它作为圈主主账号的必填开通步骤。

## 本次范围

### 做

1. Dou-Server：
   - 优化后台账号创建、编辑、查询接口。
   - 让 `tenant_owner + bound_user_id + 空 scope_circle_id` 表示“圈主主账号管理本人全部活跃圈子”。
   - 复用既有 `X-Tenant-Circle-Id` 租户切换能力。
   - 补齐圈主主账号在内容治理、通知、售后等后台接口中的多圈子范围过滤。
   - 补齐圈主主账号的默认权限预设。
2. Dou-Admin：
   - 优化后台账号表单。
   - 把圈主主账号的账号名改为远程搜索用户选择。
   - 去掉圈主主账号无意义的单圈子授权项。
   - 优化范围展示和绑定信息展示。
   - 让登录后昵称展示口径与小程序用户昵称保持一致。
3. 文档：
   - 更新使用手册、商用多租户说明和续航文档。

### 不做

1. 不新增数据库迁移。
2. 不新增新的 `scope_type` 枚举。
3. 不改小程序登录注册主链路。
4. 不把平台级提现审核权限下放给圈主账号。
5. 不重做整个后台菜单体系，只做本次账号派发和范围收口所需调整。

## 角色与权限约束

### 圈主主账号 `tenant_owner`

1. 登录名固定为绑定用户 `dxq_id`。
2. 必须绑定一个已注册小程序用户。
3. 绑定用户必须至少拥有一个活跃圈子。
4. 后台经营范围为该用户自己名下全部活跃圈子。
5. 默认预设补齐：
   - 举报查看 `report:view`
   - 举报处理 `report:process`
   - 圈主后台工作台/商品/钱包/AI 等既有权限
   - 创作者售后处理 `after_sale:manage`
6. 不包含平台级财务审核权限：
   - 不包含 `withdrawal:manage`
   - 不包含平台营收、对账、费率策略等平台权限

### 圈主子账号 `tenant_staff`

1. 仍为单圈子模型。
2. 仍要求选择授权圈子。
3. 绑定用户需是该圈成员。
4. 保持现有“指定圈子运营助手”定位。

### 圈主只读账号 `tenant_viewer`

1. 仍为单圈子模型。
2. 仍要求选择授权圈子。
3. 保持只读边界。

## 数据模型策略

本次不新增迁移，直接复用已有字段：

- `admin_users.scope_type`
- `admin_users.scope_circle_id`
- `admin_users.bound_user_id`
- `admin_users.account_type`

### 语义调整

1. `scope_type='circle' + account_type='tenant_owner' + scope_circle_id 为空 + bound_user_id 非空`
   - 表示“圈主主账号，多圈子经营模式”
2. `scope_type='circle' + scope_circle_id 非空`
   - 继续表示单圈子账号（适用于 staff / viewer）

## 后端设计

### 1. 账号创建与编辑

#### 创建圈主主账号

1. 前端提交 `bound_user_id`。
2. 服务端读取绑定用户：
   - `username = dxq_id`
   - `display_name = nickname || 抖小圈用户`
3. `scope_circle_id` 置空。
4. 校验：
   - 用户存在
   - 用户 `wechat_bound=1` 或至少是已注册小程序用户
   - 用户名下存在活跃圈子
   - `dxq_id` 对应后台账号尚未存在

#### 编辑圈主主账号

1. `username` 保持不可手改。
2. 若更换绑定用户且其 `dxq_id` 与当前账号名不一致，则拒绝，避免“账号名不是该用户 ID”的脏数据。
3. `display_name` 继续按绑定用户当前昵称回填。

### 2. 租户圈子范围

圈主主账号进入租户后台时：

1. 取 `bound_user_id` 名下全部活跃圈子。
2. 若请求头携带 `X-Tenant-Circle-Id`，则切到该圈子。
3. 未携带时默认选第一个活跃圈子。
4. 仅允许切换到本人名下圈子，禁止跨圈越权。

### 3. 平台治理接口范围收口

圈主主账号访问平台治理页时，所有列表和详情都要收口到“本人名下圈子集合”，而不是全平台：

1. 用户管理：只看本人圈子成员。
2. 圈子管理：只看本人名下圈子。
3. 资源卡治理：只看本人圈子资源卡。
4. 举报处理：只看目标属于本人圈子的举报。
5. 人工复核：只看本人圈子相关对象。
6. 售后退款：只看本人圈子售后。
7. 通知聚合：只聚合本人圈子治理与售后待办。

## 前端设计

### 1. 新建圈主主账号表单

改为：

1. 角色选择 `圈主账号` 时：
   - 显示“选择小程序用户”远程搜索框
   - 下拉展示 `dxq_id / 昵称`
   - 同时展示已拥有活跃圈子数量
2. 账号名：
   - 不再手填
   - 选中用户后自动显示为只读 `dxq_id`
3. 显示名：
   - 不再作为圈主主账号手填项
   - 改为提示“登录后右上角将展示该用户的小程序昵称”
4. 账号范围：
   - 不再展示不可点击的单选框
   - 改为只读说明文案
   - `圈主主账号：本人名下全部活跃圈子`
5. `授权圈子`、`绑定圈主`：
   - 对圈主主账号隐藏
   - 对 staff / viewer 保留

### 2. 后台账号列表

优化展示：

1. 圈主主账号显示：
   - 账号名：`dxq_id`
   - 显示昵称：当前绑定用户昵称
   - 范围：`圈主全部圈子`
2. staff / viewer 继续显示单圈子名称。

## 风险与回滚

### 风险

1. 旧代码里有不少 `scope_circle_id` 单圈子假设。
2. 若漏改平台治理接口，圈主主账号可能看到空数据或越权数据。
3. 若昵称只改创建接口不改登录态序列化，右上角仍会显示旧快照。

### 回滚点

1. 不涉及迁移，可直接回滚代码提交。
2. 若多圈子范围行为异常，可先回滚为旧的“单圈子 owner”逻辑。
3. 若前端表单异常，可先回滚 Dou-Admin；后端仍能兼容旧请求。

## 验证清单

1. 平台侧新建圈主主账号：
   - 远程搜索小程序用户
   - 自动带出 `dxq_id`
   - 成功创建
2. 圈主主账号登录：
   - 右上角显示小程序昵称
   - 未设昵称时显示 `抖小圈用户`
3. 圈主主账号切换圈子：
   - 能看到本人全部活跃圈子
   - 不能切到他人圈子
4. 平台治理页：
   - 举报、资源卡、圈子、成员、售后只返回本人范围数据
5. staff / viewer：
   - 仍需单圈子授权
   - 旧能力不回退
