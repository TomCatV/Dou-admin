# 资源卡发布表单分类与图片上传优化设计

## 背景

圈主在后台发布资源卡时，当前表单存在两个体验问题：

1. 资源卡分类下拉为空时缺少明确空态和创建入口，圈主会误以为功能异常。
2. 封面与预览图要求手填图片地址，和小程序端“选择图片后上传”的体验不一致，也容易让圈主误填不可访问或未审核的外链。

## 阶段目标

本阶段目标是把后台发布资源卡的展示字段对齐小程序现有资源卡契约，并补齐管理端图片上传闭环：

- 分类下拉展示当前经营圈子的启用分类；无分类时给出空态，并能在发布弹窗内直接进入分类管理。
- 封面图与预览图改为后台上传组件，上传成功后自动写入 `cover_url` 和 `preview_images`。
- 预览图最多 5 张，保持与小程序端一致。
- 不新增数据库字段，不改变小程序资源卡详情、购买、交付和售后链路。

## 明确不做

- 不在本阶段引入富文本编辑器。资源卡购买前展示仍采用 `preview_text + preview_images`，购买后交付仍采用 `resource_url / resource_access_code / doc_content / doc_url / code_items`。
- 不让后台复用小程序 `/api/v0.9/me/upload-asset`，避免后台 token 与小程序登录态混用。
- 不放开任意外链图片作为默认路径。后台上传后仍走服务端内容安全和 COS/本地受控路径。

## 字段契约

后台与小程序统一使用以下字段：

- `cover_url`：资源卡封面图。
- `preview_text`：购买前预览说明。
- `preview_images`：购买前预览图片数组，最多 5 张。
- `resource_url`：资源交付链接。
- `resource_access_code`：资源访问码或提取码。
- `doc_content`：购买后使用说明或卡密兑换说明。
- `doc_url`：购买后可复制打开的说明文档链接。
- `code_items`：卡密交付库存。

## P0 分类下拉可用性

- 分类接口继续使用 `/api/admin/tenant/product-categories`，由请求头 `X-Tenant-Circle-Id` 指定当前经营圈子。
- 发布弹窗内分类选择器增加“分类管理”入口。
- 分类下拉为空时展示“当前圈子还没有分类”，并提示先新增分类。
- 新建分类成功后刷新分类列表；如果发布弹窗正在打开且当前未选分类，则自动选中新建分类。

## P1 后台图片上传

### 接口

新增管理端接口：

- `POST /api/admin/tenant/upload-asset`
- 表单字段：
  - `file`：图片文件。
  - `kind`：默认 `square`。
  - `scene`：`resource_card.cover` 或 `resource_card.preview_image`。

返回：

- `url`：可公开访问的图片地址。
- `key`：COS object key，本地回退时为空。
- `provider`：`cos` 或 `local`。
- `size`、`mimetype`、`originalname`：上传文件元信息。

### 权限与租户隔离

- 接口在 `/api/admin/tenant/*` 下，先经过后台登录态、密码状态和 `requireTenantScope`。
- 仅拥有 `tenant:resource:manage` 的账号可上传。
- 上传动作作用于当前 `X-Tenant-Circle-Id` 对应圈子，并写入后台审计日志。
- 到期只读租户不能上传资源卡图片。

### 内容安全

- 服务端校验图片 MIME、文件大小与文件签名。
- 上传前执行 `checkImageFileSafety`，违规图片不入库、不返回 URL，并删除临时文件。
- COS 上传成功后注册图片异步审核任务；后续资源卡保存时继续通过 `checkImageUrlSafety` 校验 URL 是否来自受控域名和路径。

## 前端交互

- 封面图展示缩略图、上传、替换、移除，不再暴露“封面地址”输入框作为主路径。
- 预览图展示缩略图宫格，支持多选上传、删除单张、最多 5 张。
- 保存资源卡时直接提交 `preview_images` 数组，不再把 textarea 文本拆行。
- 保存按钮在图片上传中置灰，避免上传未完成就保存。

## 验收清单

- 当前圈子无分类时，发布弹窗分类下拉不再表现为空白故障，并能直接打开分类管理。
- 新增分类后，发布弹窗分类列表刷新且可选中新分类。
- 后台上传封面后，资源卡列表和编辑弹窗可看到缩略图。
- 后台上传 1-5 张预览图后，保存接口提交 `preview_images` 数组，小程序详情仍能正常展示。
- 非 `tenant:resource:manage` 账号不能上传图片。
- 到期只读租户上传图片会被拒绝。

## 回滚策略

- 前端如需止血，可回滚 `src/views/tenant/resources.vue` 的上传控件改动，临时恢复手填 URL。
- 后端上传接口为新增接口，不改变既有资源卡创建/编辑接口；回滚接口不会影响小程序现有上传路径。
