# 资源库导入解析 P0 管理后台设计

最后更新时间：2026-06-05 (Asia/Shanghai)

## 1. 页面定位

新增页面：`圈主后台 / 资源库导入`

页面不是资源站，也不是给用户直接领资源的入口。它是圈主经营后台的“资源弹药整理工具”：

1. 上传资源表。
2. 查看服务端解析结果。
3. 过滤隔离区和候选区。
4. 选择干净资源转为资源卡草稿。
5. 后续在现有“资源卡管理”中补封面、价格、说明、上架。

## 2. 菜单与权限

路由：`/tenant/resource-library`

菜单放在 `圈主后台` 下，位于“资源卡管理”之后。

| 权限 | 页面行为 |
| --- | --- |
| `circle:content:view` | 可进入页面、查看批次和明细 |
| `tenant:resource:manage` | 可上传、可转资源卡草稿 |
| 无租户上下文 | 展示“请先选择经营圈子”提示 |
| 到期/只读套餐 | 可查看，不允许上传和转换 |

## 3. 页面结构

### 3.1 顶部区域

1. 标题：资源库导入
2. 说明：上传 Excel/CSV 后，系统会自动解析、分层和隔离高风险资源。
3. 当前经营圈子选择器：复用现有 `tenantContext`。
4. 上传按钮：仅 `tenant:resource:manage` 可见。

### 3.2 批次列表

字段：

1. 文件名
2. 导入时间
3. 总资源数
4. 候选数
5. 隔离数
6. 已转资源卡数
7. 重复链接数
8. 状态

点击批次后加载明细。

### 3.3 明细筛选

筛选项：

1. 关键词
2. 风险等级：低/中/高
3. 状态：候选/隔离/已转/忽略

### 3.4 明细表格

字段：

1. 资源名称
2. 平台
3. 目标人群
4. 风险等级
5. 资源池
6. 商业化分
7. 商业化动作
8. 链接
9. 状态
10. 操作

操作：

1. 打开链接：仅作为管理员检查，不自动公开。
2. 转资源卡草稿：仅候选资源可用。
3. 已转换资源展示“去资源卡管理”。
4. 隔离资源展示风险原因，不提供转换按钮。

## 4. 转资源卡草稿弹窗

字段：

1. 资源卡标题：默认资源名称。
2. 资源卡简介：默认商业化动作或目标人群说明。
3. 价格：默认 9.90 元，圈主可改。
4. 商品分类：读取现有分类。
5. H5 展示：默认隐藏。

提交后调用：

`POST /api/admin/tenant/resource-imports/items/:itemId/convert`

成功后：

1. 提示“已生成资源卡草稿”。
2. 刷新明细状态。
3. 提供跳转到 `/tenant/resources` 的按钮。

## 5. 空态与错误态

1. 没有批次：提示“上传第一份资源表开始整理”。
2. 上传失败：显示后端返回的解析失败原因。
3. 文件格式错误：提示仅支持 `.xlsx`、`.xls`、`.csv`。
4. 高风险转换：后端拒绝后展示“隔离资源不能转为资源卡”。
5. 只读租户：展示套餐只读提示，上传按钮禁用。

## 6. 体验边界

1. 不在页面里展示大段原始 JSON。
2. 不把“隔离池”资源作为主卖点。
3. 表格里的链接只用于运营检查。
4. 页面文案强调“转草稿后仍需人工补充和上架”。

## 7. 验证清单

1. 进入页面时能加载当前经营圈子。
2. 上传 `.xlsx` 后生成批次和明细。
3. 切换圈子后批次隔离。
4. viewer 看得到列表但看不到上传和转换按钮。
5. owner/staff 能把候选资源转成资源卡草稿。
6. 转换后资源卡管理页面能看到新草稿。
7. `pnpm typecheck` 和 `pnpm build` 通过。
